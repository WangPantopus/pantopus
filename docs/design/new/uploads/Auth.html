Pantopus · Mobile Archetype · Auth Form

The shape of "let me in."

One archetype, five screens of doorway. Every Pantopus authentication screen shares the same vertical rhythm: brand lockup (mark + wordmark) → kicker overline → headline → subcopy → form fields stacked at 44px tall with label-above style → primary 48px CTA with brand halo → secondary link row → shield-check trust footer. The page is light, centered, gutter-honest. No tab bar — auth is pre-tab-bar. The error variant proves the validation vocabulary: red border, red semantic-error fill, inline message with alert-circle icon, plus an optional summary banner above the form.

Archetype slots

Six slot groups, plus the four validation states inherited from the general Form archetype (default · focus · valid · error).

brand_lockupCentered favicon mark + "Pantopus" wordmark. 48px mark + 22px wordmark on lockup-led screens; auto-shrinks on dense screens. Always present.
kickerOptional 11px primary-600 overline above the headline ("Welcome back", "Get started", "Trouble logging in", "Almost done"). Sets emotional posture.
headline24px / 700 / -0.02em letter-spacing, sentence case, centered. Names the action ("Log in to Pantopus", "Create your account", "Reset your password").
subcopy13.5px fg3, centered, max 280px wide. One line that sets context — never reads as instruction.
fields[]Stacked 44px inputs with leading Lucide icon (mail, lock), 8px radius, label-above. Passwords get an eye/eye-off toggle. Required fields show a red asterisk.
primary_ctaFull-width 48px button, primary-600 with brand-halo shadow, 12px radius. Trailing arrow on continue actions; trailing check on commit. Disabled state desaturates.
secondary_linkCentered text row below the CTA. fg3 prompt + primary-600 link ("New to Pantopus? Create account" / "← Back to log in"). 13px.
trust_footerPinned to bottom-of-screen safe area. Shield-check icon + caption — "Verified by address". Reassures the door belongs to a real platform.
terms_consentSign-up only. Checkbox + 12.5px fg2 sentence with inline Terms and Privacy links. Pre-checked never — must be deliberate.
Five frames · the canonical auth sequence
FRAME 1 · LOG IN
"Welcome back"
Email + password with show/hide eye. "Forgot password?" inline right of the password label. Primary "Log in" with arrow. Footer link to sign up.
FRAME 2 · CREATE ACCOUNT
"Get started"
Email · password · confirm. Pre-validated check on the email. Terms checkbox with inline Terms + Privacy links. Footer link back to log in.
FRAME 3 · FORGOT PASSWORD
"Trouble logging in"
Email only. Subcopy explains the reset link. Primary "Send reset link" with paper-plane icon. Footer "← Back to log in".
FRAME 4 · RESET PASSWORD
"Almost done"
New password + confirm. 3-bar strength meter sits directly under the new-password input with a label and helper. Confirm shows valid check when matched.
FRAME 5 · VERIFY EMAIL
"Check your email"
No fields — 80px primary-50 mail-check halo, the recipient address rendered bold, ghost "Resend email", underlined "Use a different email" link, plus an info pill hinting at spam/promotions.
Bonus · input error state
FRAME 6 · INPUT ERROR
"Couldn't log you in"
Same Log in skeleton, plus the validation vocabulary on full display: semantic-bg banner above the form, 1.5px red border on the failing input, semantic-bg light red fill, inline alert-circle + "Enter a valid email address."
Validation rules
  • Default → focus: border lifts to primary-600, 3px halo (rgba primary 15%), no fill change.
  • Focus → valid: border green, trailing check icon. Halo softens (8% success). Never auto-advances focus.
  • Focus → error: border 1.5px red, fill semantic-bg, inline message below with alert-circle. 11.5px / 500.
  • Error banner appears only on submit failure — never on per-field blur. Owns "Couldn't log you in" / "Couldn't create account".
  • Required indicator is a small red asterisk after the label — never used inside the placeholder.
