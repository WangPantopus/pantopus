Pantopus · Mobile Archetype · Identity Center

Profiles & privacy — the single source of truth.

One surface where the four identities sit side-by-side: Personal (account), Local (verified neighbor), Persona (public creator), Professional (hireable trade). Top-up by a "see what people see" preview so the user can always check what each viewer type sees.

Privacy by default. Profile-links toggles are all off out of the box (one is on as a deliberate user choice). Persona and Professional are opt-in — first-run users see "Not set up" placeholders, not pre-built profiles. Nothing links publicly unless the user picks it.

Archetype slots

Five slot families. identity_present drives the populated vs. first-run state.

identity_presentArray — which of [local, personal, persona, professional] are set up. Drives Frame 1 vs Frame 2 state.
profile_card{identity_type, handle, name, primary_stats_line, status} per row. Color-codes via the identity pillar.
profile_links_toggles[]Boolean per link: Local↔Persona, Persona-on-Local, Pro-to-Local. Default all off.
privacy_rows[]Chevron rows for Address sharing, Block list, Hidden interactions. Right-aligned status string optional.
disclosure_items[]"What Pantopus knows" rows — verified address, payment, identity tier, phone. Always paired with Data export.
