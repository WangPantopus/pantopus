Pantopus · A13.14 · Form (single screen)

Change password

Simple variant of the Form archetype. Three masked fields — current, new, confirm — plus a live strength meter, plus a primary "Update password" button. Save lives inline at the bottom of the body (not the top bar) because the form is short and pressing Update is the only reason the user is here. Errors land per-field with a form-level banner pinned to the top.

FRAME 1 · READY (ALL VALID)
Strong new password · Update enabled
Current verified (green check + ring), new password revealed in mono so the user can sanity-check it, all four strength rules met, confirm matches. The primary button is the live target; an info chip warns the user that other devices will be signed out.
FRAME 2 · ERROR (SUBMISSION REJECTED)
2 of 3 fields invalid · button locked
Server returned "current password wrong"; the new password is in a breach list (red ring + 2.3M-records line) and confirm doesn't match. A form-level banner counts down attempts before cooldown; a Reset-link shortcut sits under the current-password field.
