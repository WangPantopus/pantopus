Pantopus · Mobile Archetype · A13

Form (single screen)

One mobile form shell covering 14 screens — profile edit, page editor, invites, share, property details, mail-day, disambiguate. Same top-bar pattern (X + title + right action), same field rhythm, same validation language across the product.

Archetype slots

Fields that change per screen. Everything else — 44px inputs, 8px radius, 16px horizontal padding, section overlines, inline validation language — stays consistent.

slot.titleHeader text centered in the top bar (sentence case: "Invite owner", "Edit profile", "Who is this for?").
slot.right_actionOne of Save / Send / Post / Done / none. Text button, primary-600. Disabled (gray) until form valid. Omit for sticky-bottom-CTA forms.
slot.field_groups[]Array of {overline, fields[]}. Overline in UPPERCASE + 0.08em letter-spacing separates the form into scannable sections.
slot.field.typetext · email · number · slider · toggle · chips · radio · textarea · upload.
slot.field.labelAbove-input label, 12px 600 fg2. Required fields get a small * after the label.
slot.field.placeholderMuted fg4 placeholder. Prefer ghost-text examples over generic "Enter X…".
slot.field.helper11px fg3 italic under the field. Explains the "why" — what the data is used for.
slot.field.validationLive state: default → focus → valid (green + check) or error (red border + inline icon + 11px red message).
slot.sticky_ctaOptional full-width primary CTA pinned bottom. Used when the commit action needs extra weight (disambiguate, destructive).
