Delivery proof sheet

The proof-of-delivery sheet the hired worker opens from the V2 task to mark it delivered. A bottom sheet on A13 Form patterns — overline-sectioned, label-above fields, upload + textarea, inline helper copy — slid over a dimmed Task-V2 detail. One sheet, two states: compose the proof, then the submitted confirmation.

Verify-first result. A09.1 Task V2 is the gig detail screen — status pill, hero, stat strip, modules, bid list, Message + Place bid dock. It has no completion / proof-of-delivery sheet, so it’s designed here against the A13 Form vocabulary on shared tokens.
One sheet · two states
STATE 1 · ENTRY
“Confirm delivery”
Grab handle + title, a photo-proof upload grid (one photo added with a check, plus a dashed “Add” tile), an optional note field, a trust line on payment release, and a primary “Mark as delivered”.
STATE 2 · SUBMITTED
“Delivery confirmed”
Green check halo, confirmation copy, a recap card (proof thumbnail · 1 photo, note · timestamp with a double-check), and a primary “Back to task”.
