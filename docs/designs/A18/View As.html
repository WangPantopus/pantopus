src/app/identity/preview.tsx — a NEW A18 variant: instead of a wait/status, it's a preview simulation. The user picks a viewer context and watches their own profile re-render live as that viewer would see it. Privacy is the whole point of Pantopus, so this screen makes the granular reveal visible. No primary action — the screen is the output. Structure: top bar · horizontal viewer-picker chip row (Public · Persona audience · Neighbor · Connection · Household · Gig participant) · a "screen-within-screen" live render with a Viewing as… banner · privacy explainer footer.

FRAME A · POPULATED · CONNECTION VIEWER
Viewing as a connection — rich
The Connection chip is active (blue fill). Banner is info-blue. The render shows the full identity: real name, exact proximity, all three verification badges lit, mutual connections, "contact on request", plus a shared-history strip — because they've interacted before. Every visible field carries an eye glyph.
FRAME B · SECONDARY · PUBLIC VIEWER
Viewing as the public — redacted
Switch the chip to Public and the same profile collapses: name truncates to "Dana O.", avatar greys out, two badges become locked, and Location / Member-since stay coarse while Mutual connections + Contact show hatched Hidden bars. Banner flips amber ("Most details are hidden") with an eye-off strip. Same layout, different reveal — that's the simulation.
